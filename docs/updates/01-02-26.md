# Session Log - 01/02/26

## Admin Member Management - Add & Remove Members

Added functionality for admins to manage group membership for latecomers and removals.

### New Features

**Add Latecomers to Groups:**
- Unassigned accepted members now appear in a new "Unassigned Members" section
- Each unassigned member has an "Add to..." dropdown showing groups with capacity
- Confirmation modal before adding

**Remove Members from Groups:**
- Each group member now has an "Actions..." dropdown with:
  - "Move to..." options (existing functionality)
  - "Remove Member" option (new)
- Separate red trash button for remove action
- Confirmation modal warns that invitation will also be deleted

### Files Changed

**src/features/circles/circles-adminApi.js:**
- Added `addMemberToGroup(poolId, groupId, userId)` method
- Added `removeMemberFromGroup(poolId, groupId, userId)` method

**src/pages/CirclesAdmin.jsx:**
- Added state for add/remove member modals
- Added `getUnassignedMembers()` - finds accepted invitations not in any group
- Added `getGroupsWithCapacity()` - filters groups with room for members
- Added handlers: `handleOpenAddMemberModal`, `handleAddMember`, `handleOpenRemoveModal`, `handleRemoveMember`
- Added "Unassigned Members" section UI
- Changed member actions from single dropdown to separate move dropdown + remove button
- Added confirmation modals for both add and remove actions

**src/prototype.css:**
- Added `.admin-member-actions` styles for the new action layout

## Bug Fix: Unassigned Members Display

Fixed issue where assigned members were still showing in the unassigned list.

**Problem:** Was checking `member.userId` but backend returns `member.id`

**Fix:** Changed `getUnassignedMembers()` to use `member.id` when building the set of assigned user IDs.

Also fixed name display to build from `firstName` + `lastName` instead of non-existent `name` field.
