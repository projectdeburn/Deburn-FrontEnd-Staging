# Session Log - 16/02/26

## Chat Streaming Performance — Quick Wins

Long conversations caused slow AI text output due to the entire message list re-rendering on every streaming tick (~33 times/sec). Three optimizations were applied:

### 1. Memoized FormattedMessage and StreamingMessage (`src/utils/formatCoachResponse.jsx`)
- Wrapped `FormattedMessage` with `React.memo` so historical messages skip re-rendering when their `content` hasn't changed.
- Added `useMemo` inside both `FormattedMessage` and `StreamingMessage` to cache the `formatCoachResponse()` output (7+ regex operations) and only recompute when `content` changes.

### 2. Extracted memoized MessageItem component (`src/pages/Coach.jsx`)
- Moved inline message rendering into a `MessageItem` component wrapped with `React.memo` and a custom comparator (`message` + `playingMessageId`).
- Moved `getActionIcon` outside the component (pure function, no deps).
- During streaming, `streamingContent` state changes trigger Coach re-renders, but each `MessageItem` short-circuits via memo — no DOM updates for historical messages.

### 3. Debounced localStorage writes (`src/pages/Coach.jsx`)
- Replaced the immediate `localStorage.setItem` (which ran on every `messages` state change) with a 1.5-second debounced write.
- Added flush-on-unmount via refs (`messagesRef`, `conversationIdRef`) to prevent data loss if the component unmounts before the debounce fires.

## Chat Input — Textarea Conversion

Replaced the single-line `<input type="text">` with an auto-growing `<textarea>` to fix cursor/scroll position loss when switching apps.

### Changes
- **`src/pages/Coach.jsx`**: Replaced `<input>` with `<textarea rows={1}>`. Added auto-grow logic in `onChange` (resets height to `auto`, then sets to `scrollHeight`). Reset textarea height in `sendMessage` after clearing input.
- **`src/prototype.css`**: Added `resize: none`, `overflow-y: auto`, `max-height: 150px`, and `line-height: 1.5` to `.chat-input`. Changed `.chat-input-wrapper` from `align-items: center` to `flex-end` so the send button stays at the bottom when the textarea grows.

### Future Work
- Message list virtualization documented in `docs/futurePlans/chat-virtualization.md` for when conversations grow to 100+ messages.

## Voice Input — Whisper STT Integration

Replaced the browser's Web Speech Recognition API with OpenAI Whisper via a backend endpoint. The browser API had `recognition.continuous = false`, causing it to auto-stop after ~1-2 seconds of silence. Whisper records until the user explicitly stops, then transcribes server-side with higher accuracy (especially for Swedish).

### Backend (Deburn-Backend)

- **NEW `app_v2/services/media/stt_service.py`**: Created `STTService` wrapping the existing `WhisperProvider` (`common/speech_processing/whisper.py`). Validates audio size (25 MB max), empty audio, and unsupported formats.
- **`app_v2/services/media/__init__.py`**: Added `STTService` to exports.
- **`app_v2/dependencies.py`**: Registered `_stt_service` global, initialized in `init_media_services()`, added `get_stt_service()` getter — mirrors the existing TTS pattern.
- **`app_v2/routers/coach.py`**: Added `POST /transcribe` endpoint accepting multipart/form-data (`UploadFile` + `language` form field). Returns `{"success": true, "data": {"text": "..."}}`.

### Frontend (Deburn-FrontEnd)

- **`src/features/coach/coachApi.js`**: Added `transcribeAudio(audioBlob, options)` method. Posts `FormData` to `/api/coach/transcribe`, returns transcribed text.
- **`src/pages/Coach.jsx`**: Replaced `recognitionRef` (SpeechRecognition) with `mediaRecorderRef`, `audioChunksRef`, and `recordingTimeoutRef` (MediaRecorder). `toggleVoiceInput` now starts/stops MediaRecorder (webm, mp4 fallback for Safari). On stop, collects audio blob → calls `coachApi.transcribeAudio()` → auto-sends via `sendMessage()`. 120-second max recording. Reuses `isLoading` state during transcription. No new CSS or state variables.
