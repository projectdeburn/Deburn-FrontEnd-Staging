## 4. Audio Modal Improvements

### File: `/src/components/learning/AudioModal.jsx`

### 4.1 Audio Visualizer Fix
- Moved `playing` class to `.audio-visualizer` container instead of individual circles
- CSS selector `.audio-visualizer.playing .viz-orb-*` now properly triggers floating animations

### 4.2 Play Button Loading Spinner
Removed "Loading audio..." text and added inline spinner to play button:

```jsx
<button className={`audio-play-btn ${loading ? 'loading' : ''}`}>
  {isPlaying ? <PauseIcon /> : <PlayIcon />}
  <svg className="audio-loading-spinner" viewBox="0 0 48 48">
    <circle className="spinner-orb spinner-orb-1" cx="24" cy="24" r="20" fill="none" strokeWidth="2.5"/>
    <circle className="spinner-orb spinner-orb-2" cx="24" cy="24" r="15" fill="none" strokeWidth="2.5"/>
    <circle className="spinner-orb spinner-orb-3" cx="24" cy="24" r="10" fill="none" strokeWidth="2.5"/>
  </svg>
</button>
```

### 4.3 Blur-to-Focus Loading Effect
Added focus effect where content starts blurry and becomes clear when loading completes:

```jsx
<div className={`audio-player ${loading ? 'loading' : 'ready'}`}>
```

**CSS Added to prototype.css:**
```css
/* Blur-to-focus effect while loading - blur everything except play button */
.audio-player.loading .audio-visualizer,
.audio-player.loading .audio-player-title,
.audio-player.loading .audio-progress-container,
.audio-player.loading .audio-time,
.audio-player.loading .audio-control-btn {
    filter: blur(4px);
    opacity: 0.6;
    transition: filter 0.8s ease-out, opacity 0.8s ease-out;
}

/* Keep play button clear and prominent while loading */
.audio-play-btn.loading {
    filter: blur(0) !important;
    opacity: 1 !important;
    transform: scale(1.1);
    box-shadow: 0 0 20px rgba(196, 149, 106, 0.5);
}
```
