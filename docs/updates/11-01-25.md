## 02:45 (SGT) - Removing Tailwind, Using prototype.css Directly

**Decision:** Remove Tailwind CSS and use the original `prototype.css` directly for styling.

**Reason:** The initial React migration used Tailwind utility classes that approximated the original design, but did not match it exactly. This caused visual inconsistencies between the original vanilla JS frontend and the new React frontend. To ensure a true 1:1 translation, we're switching to use the same CSS classes from `prototype.css`.

**Changes to make:**

1. Copy `prototype.css` from `public/` to `front-end/src/`
2. Import `prototype.css` in the React app (replace Tailwind)
3. Update all page components to use original CSS class names:
   - Auth pages: `.auth-screen`, `.auth-container`, `.auth-header`, `.auth-form`, `.form-group`, `.form-input`, etc.
   - Main app pages: `.sidebar`, `.main-content`, `.card`, etc.
   - All components to match original HTML structure and class names
4. Remove Tailwind configuration and dependencies

**Files to update:**

- All 17 page components
- Layout components (Layout.jsx, Sidebar.jsx, Header.jsx)
- UI components (Button, Input, Card, etc.)
- index.css (remove Tailwind theme, import prototype.css)
